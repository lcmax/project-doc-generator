# 文档模板结构

> **作者获取优先级（Author Acquisition Priority）**  
> 生成文档时，作者字段按以下优先级获取：  
> 1. **当前用户**：调用 Skill 的用户标识（`$env:USERNAME` / `whoami`）  
> 2. **Git 提交作者**：目标项目最近提交作者（`git log -1 --format=%an`）  
> 3. **置空**：若以上均无法获取

> **版本差异日志（Version Diff Log）**  
> 当存在历史版本文档时，Skill 会在每个新文档末尾自动追加差异日志章节，按以下格式记录与上一版本的变更：  
> - 🟢 **新增**：新文档中存在而旧文档中不存在的内容  
> - 🟡 **修改**：新旧文档中都存在但内容有变化的部分  
> - 🔴 **删除**：旧文档中存在而新文档中已移除的内容  
> 若新旧文档无差异，则注明"与上一版本无差异"；若为全新文档，则标记为"全新文档"。

> **DOCX 输出格式（DOCX Output Format）**  
> 默认生成 Markdown（`.md`）格式文档。当用户提示词包含 `docx`、`word` 或 `.docx`（不区分大小写）时，将生成 MS Word（`.docx`）格式文档。  
> DOCX 生成策略：先按 Markdown 模板生成内容，再通过 `pandoc` 转换为 `.docx`。  
> 若系统未安装 pandoc，将回退为 Markdown 格式，并提示安装：`winget install JohnMacFarlane.Pandoc`

## 03-详细设计.md 模板

### 3.1 文档头部信息格式说明

详细设计文档支持两种版本号格式：

#### 格式一：基于 Git 提交次数
```
版本号格式：1.{yy}.{Mdd}.{git提交次数}
示例：1.26.509.156
说明：
- 1: 主版本号
- 26: 年份后两位（2026年）
- 509: 月日组合（5月9日）
- 156: Git 提交次数
```

#### 格式二：基于时间
```
版本号格式：1.{yy}.{Mdd}.{hmm}
示例：1.26.509.1430
说明：
- 1: 主版本号
- 26: 年份后两位（2026年）
- 509: 月日组合（5月9日）
- 1430: 时分组合（14:30）
```

#### 获取版本号的方法

**获取 Git 提交次数（PowerShell）：**
```powershell
$commitCount = git rev-list --count HEAD
$version = "1.{0}.{1}.{2}" -f (Get-Date).ToString("yy"), (Get-Date).ToString("Mdd"), $commitCount
```

**获取基于时间的版本号（PowerShell）：**
```powershell
$version = "1.{0}.{1}.{2}" -f (Get-Date).ToString("yy"), (Get-Date).ToString("Mdd"), (Get-Date).ToString("Hmm")
```

### 3.2 变更日志生成示例

#### 从 Git 提交记录生成变更日志

**步骤一：获取 Git 提交记录**
```powershell
# 获取最近 N 条提交记录
git log --pretty=format:"%h|%ad|%an|%s" --date=short -n 20

# 输出示例：
# a1b2c3d|2026-05-09|作者|feat: 添加用户登录功能
# e4f5g6h|2026-05-08|作者|fix: 修复订单查询bug
# i7j8k9l|2026-05-07|作者|refactor: 重构数据库连接模块
```

**步骤二：解析提交类型**

| 提交前缀 | 变更类型 | 说明 |
|----------|----------|------|
| feat | 新功能 | 新增功能特性 |
| fix | 修复 | Bug 修复 |
| refactor | 重构 | 代码重构优化 |
| docs | 文档 | 文档更新 |
| style | 格式 | 代码格式调整 |
| test | 测试 | 测试相关 |
| chore | 构建 | 构建/工具变更 |
| perf | 性能 | 性能优化 |
| ci | CI/CD | 持续集成相关 |

**步骤三：生成变更日志表格**

```markdown
## 变更日志

| 版本 | 日期 | 作者 | 变更内容 |
|------|------|------|----------|
| 1.26.509.156 | 2026-05-09 | 作者 | feat: 添加用户登录功能 |
| 1.26.508.155 | 2026-05-08 | 作者 | fix: 修复订单查询bug |
| 1.26.507.154 | 2026-05-07 | 作者 | refactor: 重构数据库连接模块 |
```

**步骤四：按类型分组展示**

```markdown
## 变更日志（按类型分组）

### 新功能 (feat)
- 添加用户登录功能 (v1.26.509.156)
- 添加订单导出功能 (v1.26.506.153)

### Bug 修复 (fix)
- 修复订单查询bug (v1.26.508.155)
- 修复数据同步异常 (v1.26.505.152)

### 代码优化 (refactor)
- 重构数据库连接模块 (v1.26.507.154)
- 优化缓存策略 (v1.26.504.151)
```

#### 自动生成变更日志脚本示例

```powershell
# generate-changelog.ps1
param(
    [int]$Count = 20,
    [string]$OutputFile = "CHANGELOG.md"
)

$commits = git log --pretty=format:"%h|%ad|%an|%s" --date=short -n $Count
$commitCount = git rev-list --count HEAD

$content = "# 变更日志`n`n"
$content += "| 版本 | 日期 | 作者 | 变更内容 |`n"
$content += "|------|------|------|----------|`n"

foreach ($line in $commits) {
    $parts = $line.Split("|")
    if ($parts.Length -eq 4) {
        $hash, $date, $author, $message = $parts
        $dateObj = [DateTime]::Parse($date)
        $version = "1.{0}.{1}.{2}" -f $dateObj.ToString("yy"), $dateObj.ToString("Mdd"), $commitCount
        $content += "| $version | $date | $author | $message |`n"
        $commitCount--
    }
}

$content | Out-File -FilePath $OutputFile -Encoding UTF8
Write-Host "变更日志已生成: $OutputFile"
```
